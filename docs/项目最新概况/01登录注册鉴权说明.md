# 用户认证系统技术文档

> **文档目的:** 记录项目登录注册和鉴权逻辑的完整实现，便于后续开发新功能时复用，防止修改影响现有认证功能。

**系统概述:** 基于 JWT + bcrypt 的用户认证系统，支持注册/登录用户认证。登录/注册 API 同时在响应体返回 token 并设置 httpOnly Cookie，服务端（Server Component）通过 Cookie 读取 token 完成鉴权。

**技术栈:** Next.js 15 + React 19 + JWT + bcryptjs + Turso SQLite

### 1.1 认证流程图

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  未登录用户   │────▶│   注册/登录   │────▶│  认证用户    │
└──────────────┘     └──────────────┘     └──────────────┘
                            │                     │
                            ▼                     ▼
                    ┌──────────────┐     ┌──────────────┐
                    │ httpOnly Cookie│    │ httpOnly Cookie│
                    │  + localStorage│    │  + localStorage│
                    │  (双端存储)    │    │  (双端存储)    │
                    └──────────────┘     └──────────────┘
```

**认证层级：**

```
┌─────────────────────────────────────────────────────────────┐
│                   API 路由层 (Node.js Runtime)               │
│  withAuth / withOptionalAuth：从 Authorization 头验证 JWT    │
│  authenticateRequest / authenticateRequestOptional           │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                  Server Component 层                         │
│  auth()：从 Cookie 读取 accessToken 并验证                   │
└─────────────────────────────────────────────────────────────┘
```

## 2. 数据库设计

### 2.1 Users 表结构

> **来源说明:** 此表定义来自 `lib/schemas/user.ts` 中的 `CREATE_USERS_TABLE`（通过 `lib/schemas/index.ts` 统一导出，`lib/db/schema.ts` 为兼容层 re-export）。注意该表不在 `INIT_SQL` 初始化列表中（INIT_SQL 仅包含 conversations 和 messages 表），Users 表在用户注册时通过 `createUser` 函数创建。

```sql
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY,           -- 用户ID (nanoid生成)
  username TEXT UNIQUE,          -- 用户名 (认证用户才有)
  password_hash TEXT,            -- 密码哈希 (bcrypt)
  is_anonymous INTEGER DEFAULT 1,-- 是否匿名用户 (0:认证用户, 1:匿名)，保留用于数据兼容，系统已不再创建匿名用户
  created_at INTEGER NOT NULL,   -- 创建时间戳
  updated_at INTEGER NOT NULL    -- 更新时间戳
);
```

### 2.2 用户类型

  is_anonymous: boolean;        // 保留用于兼容历史数据，新系统用户此字段为 false

### 3.3 认证中间件（API 路由层）

**文件:** `lib/infra/user/middleware.ts`

该模块提供两套认证机制：**包装器模式**（`withAuth` / `withOptionalAuth`）和**手动验证模式**（`authenticateRequest` / `authenticateRequestOptional`），均仅支持 JWT 认证（从 `Authorization` 请求头提取 Bearer token）。

```typescript
// 认证上下文
interface AuthContext {
  userId: string;
  isAuthenticated: boolean;
}

// 必须登录才能访问（包装器模式）
export function withAuth<T>(handler): (request) => Promise<Response>

// 可选认证（包装器模式，当前实现与 withAuth 行为一致，仅支持 JWT）
export function withOptionalAuth<T>(handler): (request) => Promise<Response>

// 手动验证 - 必须带有效 JWT
export async function authenticateRequest(request: NextRequest): Promise<AuthRequestResult>

// 手动验证 - 可选认证（仅 JWT，不支持匿名）
export async function authenticateRequestOptional(request: NextRequest): Promise<AuthRequestResult>

// 获取认证上下文（仅 JWT）
export async function getAuthContext(request: NextRequest): Promise<AuthContext>

// 检查是否为注册认证用户
export async function isRegisteredUser(userId: string): Promise<boolean>
```

**使用示例:**

```typescript
// 包装器模式：必须登录的API
export const POST = withAuth(async (request, context) => {
  // context.userId 已验证，context.isAuthenticated 为 true
  return NextResponse.json({ userId: context.userId });
});

// 包装器模式：可选认证
export const GET = withOptionalAuth<MeResponse>(async (request, context) => {
  const user = await getUserById(context.userId);
  return NextResponse.json({ success: true, user });
});

// 手动验证模式
const authResult = await authenticateRequest(request);
if (!authResult.success) {
  return NextResponse.json({ error: authResult.error }, { status: authResult.status });
}
const userId = authResult.userId;
```

### 3.8 API 端点

#### 3.8.1 登录 API

**文件:** `app/api/auth/login/route.ts`

```typescript
POST /api/auth/login

// 请求
{
  username: string;
  password: string;
}

// 响应成功
{
  success: true;
  user: { id, username, isAnonymous };
  accessToken: string;   // 15分钟有效期
  refreshToken: string;  // 7天有效期
}

// 响应失败
{
  success: false;
  error: string;
}
```

**逻辑:**
1. 验证用户名和密码必填
2. 查找用户（`getUserByUsername`）
3. 检查是否为认证用户（`is_anonymous === false`）
4. 验证密码（`verifyPassword`）
5. 生成令牌对（`generateTokenPair`）
6. 在响应体返回 token 的同时，**设置 httpOnly Cookie**（`accessToken` + `refreshToken`），供 Server Component 使用
7. 返回用户信息和令牌

**Cookie 设置：**
- `accessToken`: httpOnly, maxAge 15分钟
- `refreshToken`: httpOnly, maxAge 7天

### 4.1 认证状态管理 Hook

**文件:** `lib/hooks/use-auth.ts`

```typescript
export interface AuthState {
  user: AuthUser | null;         // 当前用户信息
  isAuthenticated: boolean;      // 是否已登录（非匿名）
  isLoading: boolean;            // 是否正在加载
  accessToken: string | null;    // 访问令牌
}

export function useAuth() {
  return {
    ...state,                    // user, isAuthenticated, isLoading, accessToken
    login,                       // 登录成功后更新状态
    logout,                      // 登出并清除令牌
    getAuthHeader,              // 获取Authorization请求头
    authenticatedFetch,         // 带自动刷新的认证请求函数
    checkAuth,                  // 检查是否需要登录
    refreshAccessToken,         // 刷新访问令牌
  };
}
```

**Token 存储（双端存储策略）：**

```typescript
// localStorage 存储（前端 API 请求使用）
localStorage.setItem("accessToken", token);
localStorage.setItem("refreshToken", token);

// httpOnly Cookie 存储（Server Component 使用）
// 由登录/注册/刷新 API 自动设置，前端无需手动操作
```

**带自动刷新的认证请求函数：**

```typescript
// authenticatedFetch: 当访问令牌过期时自动刷新并重试请求
const authenticatedFetch = useCallback(async (
  url: string,
  options: RequestInit = {}
): Promise<Response> => {
  // 1. 获取认证头（已登录用 JWT，未登录用匿名ID）
  const authHeader = getAuthHeader();

  // 2. 合并认证头到请求选项
  const mergedOptions = { ...options, headers: { ...options.headers, ...authHeader } };

  // 3. 第一次请求
  let response = await fetch(url, mergedOptions);

  // 4. 如果返回401，检查是否是令牌过期错误
  if (response.status === 401) {
    const data = await response.clone().json();
    if (data.code === "TOKEN_EXPIRED" || data.error === "访问令牌已过期") {
      // 5. 尝试刷新令牌
      const newAccessToken = await refreshAccessToken();
      if (newAccessToken) {
        // 6. 使用新令牌重试请求
        const newAuthHeader = { Authorization: `Bearer ${newAccessToken}` };
        response = await fetch(url, { ...options, headers: { ...options.headers, ...newAuthHeader } });
      }
    }
  }

  return response;
}, [getAuthHeader, refreshAccessToken]);
```

## 6. 注意事项

### 6.1 安全注意事项

1. **JWT_SECRET**: 必须设置强密钥（至少32位随机字符串），生产环境勿泄露
2. **密码强度**: 强制要求8位以上，包含字母和数字
3. **邀请码**: 注册需要邀请码，由环境变量 `INVITE_CODE` 控制
4. **Token过期**: 访问令牌15分钟过期，刷新令牌7天过期
5. **bcrypt rounds**: 使用12轮加密，平衡安全性和性能
6. **httpOnly Cookie**: 登录/注册/刷新 API 同时设置 httpOnly Cookie，防止 XSS 攻击读取 token

### 6.2 开发注意事项

1. **双端存储**: token 同时存储在 localStorage（前端请求使用）和 httpOnly Cookie（Server Component 使用），登录/注册/刷新 API 会自动同步两者
2. **缓存机制**: `getUserById` 和 `getUserByUsername` 使用 React.cache() 缓存
3. **服务端组件**: 认证Hook只能在客户端组件（"use client"）中使用
4. **环境变量**: 复制 `.env.example` 到 `.env.local` 并填写真实值
5. **路由约定**: `/chat` 已弃用，统一使用 `/agent-chat`
6. **注册限制**: 当前注册不再支持匿名用户升级，所有用户必须通过注册流程创建
