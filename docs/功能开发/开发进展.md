## 数据库初始化逻辑分离（2026-03-08）

### 已完成工作

#### 1. 移除启动时的数据库检查和迁移逻辑
- **文件**: `app/layout.tsx`
  - 移除 `DatabaseInit` 组件的导入
  - 移除 `<DatabaseInit />` 组件的使用
  - 应用启动时不再执行数据库操作

- **文件**: `components/database-init.tsx`
  - 删除该文件，不再需要在启动时执行数据库初始化

#### 2. 创建独立的数据库管理脚本
- **文件**: `scripts/init-db.ts` (新增)
  - 实现 `initDatabase` - 创建所有表结构（users、conversations、messages、folders、documents）
  - 实现 `createIndexes` - 创建所有索引
  - 实现 `migrateDatabase` - 执行字段迁移（agent_id、is_private）
  - 实现 `showDatabaseStatus` - 显示数据库状态
  - 支持命令行参数：`init` | `migrate` | `status` | `all`

#### 3. 添加 npm 脚本命令
- **文件**: `package.json`
  - `db:init` - 仅初始化表结构
  - `db:migrate` - 仅执行字段迁移
  - `db:status` - 查看数据库状态
  - `db:setup` - 执行完整的初始化+迁移流程

#### 4. 安装依赖
- **新增依赖**: `dotenv` - 用于脚本中加载环境变量

### 使用方法

```bash
# 查看数据库状态
npm run db:status

# 初始化数据库表（首次部署时）
npm run db:init

# 执行字段迁移（升级时）
npm run db:migrate

# 完整设置（初始化 + 迁移 + 状态查看）
npm run db:setup
```
